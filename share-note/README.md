# Share Note

一个基于 WebSocket 的实时共享编辑工具，支持多人同时编辑同一个文本内容，实时同步到所有连接的客户端。

## 功能特点

- 📝 **实时同步**：基于 WebSocket 实现的实时数据传输，确保所有用户看到的内容保持一致
- 📱 **响应式设计**：自动适配 PC 和手机设备，提供良好的跨设备体验
- 🔌 **自动重连**：网络断开后会在 30 秒内自动尝试重新连接
- 📊 **连接状态**：实时显示当前连接状态，让用户了解同步情况
- ⚙️ **端口配置**：支持通过命令行参数自定义服务端口

## 技术实现

- **后端**：Go 语言 + Gorilla WebSocket 库
- **前端**：原生 HTML + CSS + JavaScript
- **通信**：WebSocket 协议
- **并发控制**：使用互斥锁确保数据一致性

## 快速开始

### 前置要求

- Go 1.16 或更高版本
- Git

### 安装和运行

1. **克隆项目**

```bash
git clone https://github.com/yourusername/network-toolbox.git
cd network-toolbox/share-note
```

2. **安装依赖**

```bash
go get github.com/gorilla/websocket
```

3. **运行服务**

```bash
go run main.go
```

默认会在 8080 端口启动服务，你可以通过浏览器访问 `http://localhost:8080` 来使用。

### 自定义端口

如果你需要使用不同的端口，可以通过 `-port` 参数指定：

```bash
go run main.go -port=3000
```

## 使用方法

1. 打开浏览器访问服务地址（如 `http://localhost:8080`）
2. 在文本框中输入内容，所有连接的用户都会实时看到更新
3. 页面会显示当前连接状态，确保数据同步正常
4. 可以在多个浏览器标签页或不同设备上打开同一地址，体验实时同步效果

## 核心功能实现

### WebSocket 连接管理

- 建立 WebSocket 连接并处理客户端消息
- 维护在线客户端列表，实现消息广播
- 处理连接断开和错误情况

### 数据同步机制

- 当一个客户端发送消息时，服务器会：
  1. 更新服务器端的内容
  2. 将消息广播给所有连接的客户端
  3. 确保所有客户端显示相同的内容

### 响应式设计

- 使用 CSS 媒体查询适配不同屏幕尺寸
- 在 PC 端提供固定高度的编辑区域
- 在手机端自动适配屏幕高度，提供更好的触摸体验

## 注意事项

- 本项目为演示用途，未实现数据持久化，服务重启后数据会丢失
- 未实现用户认证和权限管理，任何人都可以编辑内容
- 建议在局域网或信任的网络环境中使用
